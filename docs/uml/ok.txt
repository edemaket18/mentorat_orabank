Diagramme des routes API REST

/api/auth
├── POST    /register           => Création de compte
├── POST    /login              => Connexion
├── GET     /me                 => Données utilisateur connecté

/api/profiles
├── GET     /                  => Liste des profils
├── GET     /:id               => Profil par ID
├── PUT     /:id               => Modifier un profil

/api/students
├── GET     /                  => Liste des étudiants
├── GET     /:id               => Étudiant par ID

/api/mentorships
├── GET     /                  => Liste
├── POST    /                  => Créer une demande
├── PUT     /:id/status        => Modifier statut

/api/experiences
├── GET     /                  => Voir les expériences
├── POST    /                  => Partager une expérience

/api/skills
├── GET     /                  => Liste des compétences
├── POST    /                  => Ajouter

/api/reports
├── POST    /                  => Signaler un utilisateur
├── GET     /                  => Voir les signalements (admin)
/api/users
├── GET     /                  => Liste des utilisateurs
├── GET     /:id               => Utilisateur par ID
├── PUT     /:id               => Modifier un utilisateur
├── DELETE  /:id               => Supprimer un utilisateur
/api/notifications
├── GET     /                  => Voir les notifications
├── POST    /                  => Créer une notification
/api/notifications/:id
├── PUT     /:id               => Marquer comme lu
/api/notifications/:id
├── DELETE  /:id               => Supprimer une notification
/api/notifications/:id
├── GET     /:id               => Voir une notification par ID
 

Diagrammes de séquence
a) Inscription d’un utilisateur

 Utilisateur -> Frontend : Remplit formulaire /register
Frontend -> Backend : POST /api/auth/register
Backend -> MongoDB : Vérifie si l’utilisateur existe
Backend -> MongoDB : Crée User et Profile
Backend -> Frontend : Retourne token + infos utilisateur
Frontend -> Utilisateur : Affiche message de succès


b) Connexion d’un utilisateur

Utilisateur -> Frontend : Remplit formulaire /login
Frontend -> Backend : POST /api/auth/login
Backend -> MongoDB : Vérifie les identifiants
Backend -> Frontend : Retourne token + infos utilisateur
Frontend -> Utilisateur : Affiche message de succès



c) Création d’un profil
Utilisateur -> Frontend : Remplit formulaire /profiles
Frontend -> Backend : POST /api/profiles
Backend -> MongoDB : Crée le profil
Backend -> Frontend : Retourne les infos du profil
Frontend -> Utilisateur : Affiche message de succès


d) Création d’une demande de mentorat
Utilisateur -> Frontend : Remplit formulaire /mentorships
Frontend -> Backend : POST /api/mentorships
Backend -> MongoDB : Crée la demande de mentorat
Backend -> Frontend : Retourne les infos de la demande
Frontend -> Utilisateur : Affiche message de succès

e) Partage d’une expérience
Utilisateur -> Frontend : Remplit formulaire /experiences
Frontend -> Backend : POST /api/experiences
Backend -> MongoDB : Crée l’expérience
Backend -> Frontend : Retourne les infos de l’expérience

f) Signaler un utilisateur

Backend -> Frontend : Retourne les infos du signalement

g) Consultation des notifications

Frontend -> Utilisateur : Affiche les notifications


h) Marquer une notification comme lue

 Demande de mentorat

 Utilisateur (mentee) -> Frontend : Formulaire demande
Frontend -> Backend : POST /api/mentorships
Backend -> MongoDB : Crée une entrée Mentorship
Backend -> Frontend : Confirmation
Frontend -> Utilisateur (mentee) : Affiche confirmation
Mentor -> Frontend : Notification nouvelle demande
Frontend -> Backend : GET /api/notifications
Backend -> MongoDB : Récupère notifications
Frontend -> Frontend : Affiche notifications
